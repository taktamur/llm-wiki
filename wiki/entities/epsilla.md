# epsilla

ベクトル DB / Semantic Graph 系のエンタープライズ検索プロダクトを提供する企業。[[blog-epsilla]] の著者。

本 wiki の文脈では、[[llm-wiki]] を「個人用途としては妥当だがエンタープライズには不十分」と評し、代替として [[semantic-graph]] を提示した主体として参照する。

## 立場

- RAG 1.0（任意チャンク + ベクトル）は時代遅れと主張
- Karpathy の LLM Wiki の **構造化・相互接続・人間可読** という 3 原則は評価
- ただし **RBAC / 監査 / スケール / セキュリティ**の観点でエンタープライズ要件を満たさないと指摘
- 自社製品（semantic graph）がこれらを満たす、というポジショニング

## 評価メモ

自社製品への誘導バイアスあり。批判の骨子は [[rohitg00-v2-gist]] の **Privacy and governance** セクションと重なる。本 wiki は個人用途なので直接の影響は小さい。
